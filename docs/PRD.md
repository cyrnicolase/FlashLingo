# 程序员英语词汇背诵 APP PRD

## 1. 产品概述

### 1.1 产品定位

面向程序员的英语词汇背诵工具，APP 内置初始词库，支持 API 增量同步词汇，采用「闪卡 → 测试」两阶段学习模式，帮助程序员高效记忆技术词汇。

### 1.2 目标用户

- 有英语词汇学习需求的程序员
- 需要阅读英文技术文档的开发者

### 1.3 核心价值

| 价值 | 说明 |
|------|------|
| 专注场景 | 内置程序员常用词汇，覆盖技术术语 |
| 高效记忆 | 闪卡初识 + 测试强化，双重记忆效果 |
| 自主可控 | API 增量同步，用户可自定义扩展词库 |

### 2.3 数据同步架构

```
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│  远程词库    │ ←──────→ │   APP 端    │ ←──────→ │  本地数据库  │
│  (服务器)    │  HTTP   │  (同步服务)  │  Room   │  (SQLite)   │
└─────────────┘         └─────────────┘         └─────────────┘
                              │
                              ▼
                       ┌─────────────┐
                       │  内置词库    │
                       │ (assets/)   │
                       └─────────────┘
```

**同步策略：**

| 时机 | 行为 |
|------|------|
| 首次安装 | 使用内置词库 |
| 每次启动 | 调用 sync API 增量同步 |
| 同步成功 | 更新本地数据库 |
| 同步失败 | 使用本地已有数据 |

| 场景 | 闪卡阶段 | 测试阶段 |

| 新词>0, 复习=0 | 完成新词闪卡 | 完成新词测试后结束 |

| 答错 | 选项变红 + 正确答案变绿 + 鼓励语（随机） + 显示「下一题」按钮 |

**鼓励语池（答错时随机显示）：**

| 鼓励语 |
|--------|
| 加油！再想想！ |
| 别灰心，继续加油！ |
| 这词有点难，记住了！ |
| 继续保持，你能行！ |
| 温故知新，再接再厉！ |
| 差一点就对了！ |
| 熟能生巧，多练几遍！ |
| 记忆需要反复，加油！ |

**选项生成规则：**

| 规则 | 说明 |
|------|------|
| 来源 | 同字母分组的其他单词 |
| 数量 | 4 个选项 |
| 约束 | 确保选项语义不同 |
| 补充 | 同组词不足时从相邻组补充 |

## 5. 学习算法

### 5.1 每日新词分配

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 每日新词数 | 20 | 每日新增学习词汇量 |

**分配逻辑：**

```
1. 从未学习过的词 → 加入今日新词池
2. 每日新词上限 20 个
3. 新词按导入时间排序（先导入先学习）
```

### 5.2 复习任务分配

| 条件 | 规则 |
|------|------|
| nextReviewAt ≤ 今日 | 加入今日复习任务 |
| 优先顺序 | 错词 > 待复习词 > 已掌握词 |

### 5.3 掌握判定

| 场景 | 规则 |
|------|------|
| 测试模式答对 | correctStreak +1 |
| 测试模式答错 | correctStreak 重置为 0 |
| 连续答对 2 次 | isMastered = true，下一轮复习在 3 天后 |
| 错词连续答对 3 次 | 从错词本移除 |

### 5.4 复习间隔计算

| 场景 | nextReviewAt |
|------|--------------|
| 答错 | now + 1 天 |
| 首次答对 | now + 1 天 |
| 连续答对第 2 次 | now + 3 天 |
| 已掌握词答错 | now + 1 天，isMastered = false |

## 6. API 接口

### 6.1 词汇同步

**接口地址：** `GET /api/v1/words/sync`

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| since | Long | 否 | 上次同步时间戳，为空则返回全量 |

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "words": [
      {
        "word": "abstract",
        "phonetic": "əbˈstrækt",
        "meaning": "抽象的",
        "example": "abstract class"
      }
    ],
    "deleted": ["deprecated-word"],
    "syncTimestamp": 1712304000000
  }
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| words | 新增或变更的单词 |
| deleted | 已删除的单词列表 |
| syncTimestamp | 本次同步时间戳，客户端存储用于下次请求 |

**同步流程：**

```
APP 启动
    │
    ▼
读取本地 syncTimestamp
    │
    ▼
调用 GET /api/v1/words/sync?since={timestamp}
    │
    ├─── 成功 ───┐
    │            │
    ▼            ▼
解析 words    解析 deleted
    │            │
    ▼            ▼
新增/更新    删除本地词汇
本地数据库    数据库
    │            │
    └────┬───────┘
         │
         ▼
  存储 syncTimestamp
         │
         ▼
    同步完成
```

### 6.2 词汇导入（预留）

**接口地址：** `POST /api/v1/words/import`

**请求示例：**

```json
{
  "words": [
    {
      "word": "abstract",
      "phonetic": "əbˈstrækt",
      "meaning": "抽象的",
      "example": "abstract class"
    }
  ]
}
```

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "imported": 100,
    "duplicates": 5
  }
}
```
